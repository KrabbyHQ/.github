# Security Policy

This repository serves as the organizational base for the Krabby ecosystem/project.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Security is a top priority for the Krabby Engineering Team. If you discover a vulnerability or a security concern related to our organizational assets or engineering standards, please report it privately via:

📧 okpainmoandrew@gmail.com

When reporting, please include:

- A clear description of the vulnerability or concern.

- Steps to reproduce or a description of the affected asset/document.

- Potential impact assessment.

We aim to acknowledge receipt within 48 hours and will keep you informed throughout the investigation and remediation process.

We follow responsible disclosure practices and kindly ask that you allow us reasonable time to investigate and address the concern before disclosing it publicly.
